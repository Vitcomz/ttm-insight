1.- Acceso al Sistema
     
Para el control de acceso a usuarios, el sistema dispone de una interfase que verifica el nombre de usuario y su contraseña. Esta autenticacion tiene un tiempo de duración de 12 Horas aprox., este tiempo puede ser modificado con el administrador del sistema a libre elección. Transcurrido este tiempo el usuario deberá volver a ingresar su contraseña para acceder al sistema.

3.- Equipos (Conceptos y Administración).
 

Los equipos estan divididos en sub equipos y a su vez estos se dividen en otros subequipos obteniendo 3 niveles de detalle por equipo. Por ejemplo tenemos que un Auto se podria dividir en motor y chasis, a su vez el Motor puede tener sub equipos como el Alternador, la secuencia de esto seria Auto->motor->alternador.

Ver Ejemplo.

Cada sub equipo se administra de forma unica, pero forman parte de un equipo principal, asi podremos asignar alarmas de mantención independientes para cada sub equipo.

Los atributos a considerar en los equipos son :

CODIGO : Es de tipo alfanumerico lo cual nos permite crear codigos inteligentes.

DESCRIPCION : Nos servirá para describir el equipo en todo el sistema, es importante una descripcion lo mas representativa posible.

GRUPO : Identifica a que grupo de equipo pertenece, este nos servira para asignar un horometro del grupo seleccionado.

ESTADO : Los estados que puede tener un equipo son : Activo y En Mantención.

PROVEEDOR : Indica el proveedor que distribuye la maquinaria.

MARCA : Marca de un equipo.

HOROMETRO : Especifica el horometro que controlara el equipo actual, dentro de un equipo complejo pueden existir varios horometros como el Diesel, Hoist, Etc. en este campo se debe asignar el que controlara el equipo.

FECHA : Fecha de Ingreso en el sistema.

UBICACIÓN : Lugar de Trabajo del Equipo.

HOROMETRO ACTUAL : Indica el estado actual del Horometro, en este ejemplo el horometro tiene 194 horas desde su activación.

VENCIMIENTO HOROMETRO : Indica cuando vencerá el horometro, en este ejemplo no se ha especificado este vencimiento, por lo tanto se muestra en color rojo las horas vencidas 194.

PERIODO DE MANTENCIÓN : Indica cual será el lapso de tiempo, que deberá transcurrir para realizar la proxima mantención.

ALARMA DE MANTENCIÓN : Nos indica cuantas horas antes queremos que el sistema nos advierta que el equipo necesita mantención.

ULTIMA MANTENCIÓN : Fecha de la ultima mantención realizada al equipo, esta se actualizará cada vez que se ejecute una OT.

4.- Horometros.
 

La administración de alarmas se hace a travez del horometro el cual es ingresado por el administrador del sistema al momento de crear el equipo. Previamente debe crearce un grupo de equipos, este concepto nos ayudara a agrupar todos los equipos para luego crear los correspondientes horometros del grupo.

Entonces tendremos Grupo G1 (Grua Gottwald) -> Horometros (Diesel, Hoist, Luffing, etc)

Una vez creados los horometros podemos crear un equipo y asignar su grupo y horometro.

Tambien existe un modulo para la asignación masiva de horometro, esta se encuentra en el menu de Administración->Equipos->Horometros.

6.- Bitacora.
 
Para el registro diario de las actividades se encuentra el modulo de bitacoras, en este podemos registrar dos tipos de actividades: Mantención o Servicios este nombre esta referido al tipo de trabajo que se ejecuta. Una actividad puede generar un incidente el cual nos generará una OT de mantención correctiva, en la interfase principal podemos encontrar filtros por equipo, turno, responsable y periodos de tiempo.
 
Para ingresar una bitacora debemos presionar sobre cualquier iconos , en esta ventana debemos ingresar el equipo asociado y la observación del trabajo o incidente a registrar, el correlativo será generado de forma automatica, las fechas y horas serán obtenidas desde el sistema.

Los atributos a considerar son :
EQUIPO : Indica el equipo asociado a la observación o incidente, este campo es opcional, debido que la bitacora esta para registrar cualquier actividad relevante y no necesariamente puede tener un equipo asociado.

INCIDENTE : Nos indicara que huvo un incidente y este debera solucionarce con una mantencion correctiva, este estado de incidente se verá visualizado en el modulo de mantenciones correctivas y no cambiara hasta que se ejecute la mantención correspondiente.

PRIORIDAD : Cuando existe un incidente grave se puede dar prioridad de ejecución, este checkbox nos facitara el seguimiento. ESTADO : Los estados que puede tener un equipo son : Activo y En Mantención.

TIPO : Esta referido al tipo de trabajo asociado, Mantención o Servicio, cuando la bitacora es de servicio se deberá llenar los datos de Cinta, Operadores, Nave, Report y Daños.

OBSERVACION : Este texto tiene una capacidad maxima de 750 caracteres.

DAÑOS : Si durante el turno ocurrio un daño, este deberá se debera registrar para su posterior gestion.

REPORT : Nos servirá para asociar la bitacora al nro de report del servicio.

CINTAS Y OPERADORES : En estos campos se debe registrar el personal y cintas transportadoras utilizadas.

7.- Mantención Preventiva.
 

Todos los equipos que tengan vencidos o por vencer su horometro contador serán desplegados en esta ventana, el coordinador podra asignar un color de prioridad siendo el color rojo una prioridad alta y color amarillo indica prioridad baja. Los equipos que esten por vencer, serán mostrados según las horas que se ingresarón el el campo horometro alarma del modulo de equipos.

Este listado será ordenado desde el equipo mas vencido en forma decendiente, el coordinador de mantención deberá decidir que equipo deberá ser mantenido primero, generando una OT para este.
Para generar una OT debemos presionar sobre el icono :
Solo nos bastará asignar un responsable, la fecha y turno de realización.

8- Mantención Correctiva.
 
Este modulo entregara un listado de todos los incidentes ingresados en bitacora, esta listado estara ordenado por los dias transcurridos desde que se detecto el incidente, el coordinador de mantención deberá generar la correspondiente OT asignando un responsable y fecha para realizar el trabajo, al igual que las mantenciones preventivas este modulo tiene una lengueta con la consulta de las OTs ya generadas.

9.- Orden de Trabajo O.T.
 

La Orden de Trabajo es un modulo que permitira al supervisor ingresar las actividades de mantención efectuadas, existen dos tipos de orden de trabajo la correctiva que se origina a partir de un incidente y la preventiva que se origina atravez del vencimiento del horometro del equipo.

Cada supervisor tendra un listado de sus propias OTs pendientes, estas desaparecerán una vez ejecutadas.

Para las OT de tipo preventiva se adjuntaran en la OT de forma automatica las actividades de mantención del equipo, para las OT correctivas es usuario deberá ingresar las actividades realizadas para solucionar el problema.

 

Los atributos a considerar son :


FOLIO : Correlativo que se genera de forma automatica.

EQUIPO : Equipo al cual se le realiza la mantención, cabe destacar que todas las OT estan asociadas a un EQUIPO, no puede existir un OT por si sola.

COMPONENTE : Es el elemento basico donde se produjo la falla en caso de ser una OT correctiva, esto nos ayudará a realizar gestion sobre una maquina en el trascurso del tiempo.

FECHA PROGRAMADA : Indica la fecha en que el coordinador de mantención programo la ejecucion de este.

ESTADO : Los estados que puede tener una OT son : Programada, Pendiente, Anulada y Ejecutada.

TURNO : Indica el turno cuando fue ejecutado.

OBSERVACION : Este texto tiene una capacidad maxima de 750 caracteres.

RESPONSABLE : Responsable de ejecutar la OT.

EJECUCION : Periodo que demorá en realizar la OT., en este caso se debe ingresar Fecha Inicio y Fecha Final.

TIPO : Indica el tipo de OT.
